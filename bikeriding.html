📌 프로젝트 개요: BikeRiding

프로젝트 기간

2025년 4월 ~ 2025년 5월

기술 스택

HTML, CSS, JavaScript, React, Node.js, Express, MongoDB

배포 환경

AWS EC2 (Ubuntu 20.04), Nginx, PM2

🎯 프로젝트 목표

‘자전거 라이딩’을 입문하거나 취미로 즐기는 초보자들이 보다 편하게 코스를 알아보고, 지역 및 원하는 조건에 맞는 라이딩 크루를 쉽게 찾을 수 있도록 도우며, 같은 취미를 가진 사람들이 정보 공유할 수 있는 웹 개발

🛠️ 주요 기능

  • 회원 관리: CRUD, 프로필 사진 업로드, 로그인/로그아웃 등 기본 사용자 관리
    - Spring Boot 기반으로 회원 등록, 수정, 삭제 등의 기능을 구현하고, MySQL에 사용자 정보를 저장했습니다.
    - 로그인/로그아웃 기능은 Spring Security 없이 직접 구현했으며, 세션을 통해 로그인 상태를 관리합니다.
    - 프로필 사진은 MultipartFile을 이용해 업로드하고, 서버 로컬 디렉토리에 저장한 후 DB에는 경로만 저장합니다.
    - 비밀번호는 BCryptPasswordEncoder를 사용해 안전하게 암호화하여 저장합니다.
  • 코스 관리: CRUD, 찜(즐겨찾기) 기능
    - 사용자 맞춤 추천을 위해 코스 정보는 개별 테이블로 관리하며, Spring Boot로 CRUD 기능을 제공합니다.
    - 사용자는 원하는 코스를 즐겨찾기(찜)할 수 있으며, Ajax 요청을 통해 실시간으로 UI에 반영됩니다.
    - 찜 기능은 사용자와 코스 간의 매핑 테이블로 구현되어 있으며, 중복 찜 방지를 위해 제약 조건을 설정했습니다.
  • 크루 관리: CRUD
    - 사용자가 직접 크루를 생성하고, 크루에 가입 및 탈퇴할 수 있는 기능을 구현했습니다.
    - 크루 정보는 별도의 테이블에서 관리되며, 리더 정보, 활동 지역, 가입 조건 등을 함께 저장합니다.
    - Ajax를 사용해 크루 등급 변경 및 가입 승인 처리 등을 페이지 새로고침 없이 비동기로 처리합니다.
  • 채팅 기능 (WebSocket): 실시간 소통
    - Spring WebSocket과 STOMP를 이용해 실시간 채팅 기능을 구현했습니다.
    - 채팅 클라이언트는 SockJS와 Stomp.js를 사용하며, 브라우저에서 실시간 메시지를 송수신할 수 있습니다.
    - 각 크루별 채팅방이 분리되어 있어, 사용자들은 자신의 크루 내에서만 실시간 대화를 나눌 수 있습니다.
  • 크루별 게시판 기능: CRUD, 댓글 작성, 페이지네이션
    - 각 크루마다 전용 게시판이 있으며, 게시글 작성, 수정, 삭제 기능을 제공합니다.
    - 게시판은 Spring Boot와 MySQL 기반으로 구현되었으며, 댓글은 게시글과 연관된 별도 테이블에서 관리됩니다.
    - 페이지네이션은 Spring Data JPA의 Pageable 인터페이스를 활용하여 구현되었습니다.
  • 비동기 통신 (Ajax): ID/닉네임 중복 체크, 크루 등급 변경, 신고 상태 변경
    - jQuery 기반 Ajax를 사용하여 ID/닉네임 중복 확인을 실시간으로 처리할 수 있게 구현했습니다.
    - 관리자 기능에서 크루원 등급 변경 및 신고 상태 변경도 Ajax로 처리하여 UX를 개선했습니다.
    - 서버에서는 REST API를 통해 요청을 처리하고, JSON 형식으로 결과를 반환합니다.
  • 반응형 디자인: 다양한 디바이스에서 최적화된 UI 제공 (미흡)
    - Bootstrap을 활용하여 기본적인 반응형 UI를 구성했으며, 모바일/태블릿에서도 콘텐츠가 깨지지 않도록 조정했습니다.
    - 일부 페이지에서는 CSS 미디어 쿼리를 직접 사용하여 레이아웃을 조정했으나, 완전한 반응형 구현은 미흡한 상태입니다.

📝 회고 및 느낀 점

이번 프로젝트는 저에게 첫 장기 협업 프로젝트 경험이었으며, 약 한 달간 진행되었습니다. 수업을 통한 팀 구성으로 체력적·심리적 부담으로 인한 팀원 이탈이 있었으나, 남은 인원들이 끝까지 완성한 경험은 큰 의미로 남았습니다.

프로젝트를 통해 배운 점:

  • 협업의 어려움과 중요성: 의사소통 부재와 작업 분배 문제에서 소통의 중요성을 다시 깨달음
  • 심리적 동기 부여의 필요성: 부족함을 느끼는 팀원들을 응원하고 도와주는 분위기의 중요성 인지
  • 꾸준함의 힘: 힘든 상황에서도 포기하지 않고 끝까지 완수하는 자세 체득

단순 기술 성장을 넘어, 좋은 개발자의 태도와 협업 자세를 배운 소중한 기회였으며, 앞으로도 책임감 있고 소통하는 개발자가 되겠습니다.
